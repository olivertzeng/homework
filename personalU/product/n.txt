# 多元表現綜整心得

從接觸摺紙開始，我就對「把東西做出來」這件事有強烈的興趣。小學四年級我從 JeremyShaferOrigami 的 YouTube 頻道學會 Jeff Beynon 設計的 Spring into Action，一個需要精準預摺與壓縮才能成型的螺旋彈簧結構。國中生活科技課要交作品時，我決定挑戰用全開紙重新製作，最終不僅完成了成品，還自行設計木製展示架搭配 IR 感應燈條。這個經驗讓我發現：我喜歡的不只是「學會一個技法」，而是「把學到的東西延伸，然後想辦法呈現出來」。

這個習慣延伸到了程式設計。國中二年級暑假因為好奇心接觸了 iOS 越獄與 Reddit，隔年靠著 Arch Wiki 獨立完成 Arch Linux 的安裝與配置。父親從事底層 WiFi 驅動程式設計，他帶我入門了 git 與 vim，但後續的探索完全出於自發——我甚至反過來幫他的手機安裝 AltStore 以使用無廣告的 YouTube。

高中加入電算社後，我用 Python 的 pygame 與組員合作開發了一款遊戲，第一次體會到與人溝通的難度不亞於寫程式本身。隨後我注意到許多開源專案缺乏正體中文支援，起初逐檔手動翻譯，但很快意識到這個流程可以自動化——於是我寫了 replace.sh，一支 1800 行的自動化腳本，透過 OpenCC 搭配自訂字典，將簡體中文轉換為自然的臺灣用語，甚至能處理 PNG 內嵌的 metadata 與檔名。靠著這套工具，我在 topgrade-rs、yay、BleachBit 等專案累計了 145 個被合併的 Pull Request。

我也開發了 yt.py，一支多執行緒的 YouTube 播放清單下載器，處理音樂標籤、封面嵌入與 SponsorBlock 片段追蹤，並串接到我自架的 Navidrome 音樂串流伺服器。這只是我在 Arch Linux 主機上運行的十餘項 Docker 服務之一，其餘包括廣告過濾、照片備份與檔案同步等。

在技術之外，自從幼稚園中班我透過鋼琴與手風琴的學習（奧福八級、史坦巴哈九級）培養了對節奏與結構的敏感度。2023 年我將重心轉向程式設計，但對音樂的興趣並未消失——我仍然透過音遊與自架的串流服務持續接觸音樂。首次應試 TOEIC 即取得 960 分，APCS 程式識讀四級、程式實作三級，這些成績反映的不只是考試準備，而是長期自主學習累積的結果。

回顧高中三年，我做的每一件事都遵循同一個模式：發現一個有趣的東西、學會它、把它延伸、然後分享出去。從全開紙的摺紙到 1800 行的自動化工具，規模不同，但驅動力始終相同。我想繼續這個循環。
