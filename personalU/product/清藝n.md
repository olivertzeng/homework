# 多元表現綜整心得

## 從紙張到程式碼：同一套方法論的不同媒材

我的多元表現橫跨實體創作、程式開發與跨國交流三個領域，但貫穿這些經驗的核心邏輯只有一個：看到有趣的東西就動手做，做完拿去跟人討論，從討論中找到下一步。

## 實體創作與國際交流

國二時我將原本掌上尺寸的 Spring into Action 摺紙結構放大成懸掛式燈具，自行設計木製支架並加裝紅外線感測 LED 互動裝置。高二時透過開源視訊軟體 Jitsi 與這個結構的原創者、摺紙藝術家 Jeremy Shafer 進行了一小時的英語一對一課程，他對放大版的想法表示驚艷，並因為得知我會拉手風琴而額外教了我摺紙手風琴的技法。這次交流讓我確認，把想法做成實體再拿去跟該領域的人討論，是最有效的學習方式。

## 程式開發與開源貢獻

同樣的「動手做」衝動延伸到程式領域。我開發了 replace.sh 解決開源社群的簡繁轉換問題，累計 145 個 PR 被國際開源專案合併。個人網站 7279744.xyz 的漸層配色與圓角卡片視覺風格是我自己設計的，並製作了繁中、簡中、英文三語版本。

## 共通的方法論

摺紙時我先畫設計圖再動手，寫程式時我先規劃架構再寫 code。摺紙用海報紙而非專業用紙是材料的妥協，寫 Bash 而非 Python 是環境的妥協。在限制條件下用最精簡的結構達成最大的表現力，這個原則在兩個領域完全相通。APCS 觀念 4 級與實作 3 級、TOEIC 960 分則分別驗證了我在程式邏輯與英語溝通上的基礎能力。
