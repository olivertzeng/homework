# 就讀動機（P）：國立臺灣科技大學資通科技國際學士學位學程

## 一、從自架伺服器到資通系統的全貌理解

我高中期間在 Arch Linux 環境下，利用 Docker 自行部署並維護了包含 AdGuard、Navidrome、Immich、Syncthing 等十餘項網路服務，並導入 Tailscale 建立跨裝置的虛擬私人網路。這段經歷讓我對 DNS 解析、反向代理與容器間通訊有了實務層面的理解。

然而，我目前的知識完全建構在「自己摸索」上。我能讓服務跑起來，卻無法系統性地解釋封包從應用層到實體層的完整路徑，也不太清楚 Tailscale 的 WireGuard 之間的關係。在缺乏網路協定與通訊架構的正規訓練下，讓我的除錯方式停留在「改設定檔，重啟，看有沒有好」的層次。

## 二、為何選擇台科大資通科技國際學程

我曾親自前往台科大校園實地探訪。在圖書館中翻閱到資工系戴文凱教授指導的碩士論文《基於深度學習之音樂遊戲拍譜自動生成》，身為一個投入音樂遊戲的玩家，看到深度學習能應用於自動生成遊戲拍譜，讓我感受到台科大將技術研究與生活應用結合的文化。

IICT 的課程規劃涵蓋了我目前的知識缺口：大一的數位邏輯設計與計算機程式能讓我從底層重建基礎；大二的演算法、資料結構與計算機網路概論，是我在自架 Docker 服務時反覆碰壁卻無法系統性理解的核心領域；大三的計算機組織與微算機概論則能補足硬體底層的空白。

更關鍵的是，IICT 將「國際研修」列為四年級必修。從大一學生參加第四屆電資實務應用國際研討會（IPAS）的心得中，我得知學程新生即能以見學組身分前往韓國延世大學，實地參觀 6G 機械手臂與衛星通訊實驗室。這種從入學第一年就啟動的國際視野培育，結合我既有的 TOEIC 960 分與 GitHub 跨國協作經驗，讓我能在國際化的學術框架中將實戰能力進一步深化。

大學期間，我將透過七選四專業選修深耕軟體工程與作業系統，並主動參與 IPAS 國際研討會與大四的國際研修專案。畢業後預計投入雲端架構或 DevOps 領域，將自架伺服器的實戰經驗與大學四年的正規訓練結合，開發能服務廣大技術社群的網路基礎設施。
